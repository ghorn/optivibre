# Linear SSIDS/SPRAL Source-Difference Audit

This audit lists source-level implementation differences between native
SPRAL/METIS and the pure-Rust linear SSIDS lane. It is a work queue, not a
branch ledger. Native SPRAL and native METIS are test oracles only; production
`ssids-rs` must stay native-free.

Related evidence:

- `docs/spral-matching-scaling-port.md`
- `docs/spral-ssids-parity-flow.md`
- `docs/native-spral-parity-manifest.md`
- `docs/ssids-glider-performance-source-compare.md`

Priority legend:

- `P0`: next correctness or major performance parity item.
- `P1`: important parity or evidence-hardening item after P0.
- `P2`: known lower-risk difference; keep visible and fix if it becomes hot.
- `P3`: accepted deviation or out-of-hot-path difference; monitor.
- `Accepted`: source-backed and test-backed deviation with no current action.

## Numeric Factor

Latest small-leaf TPP update: the native kernel shim now reports release
native/Rust `ldlt_tpp_factor` medians on deterministic glider-shaped panels. On
synthetic size `33/48/64` fixtures Rust is faster than the source-built native
shim, so the remaining glider factor delta is not proven to be an isolated
rank-update helper problem. A pointer-hoisted Rust loop rewrite preserved the
native prefix trace but regressed the real glider profile, so it was rejected as
a Rust-only micro-optimization rather than a source-shape convergence step. The
debug-only exact-panel capture now records real glider small-leaf TPP panels
before factorization; the current replay captures `235` panels, largest
`m=71,n=57,ldl=72,hash=0x5c45f34fbf8835fc`.

## Fix Order

1. `P0`: optimize the now-active source-shaped small-leaf path without changing
   observable output: TPP pivot/factor loops first. Small-leaf output merging
   and non-root contribution packing are now closed.
2. `P1`: re-run the parallel safety matrix and side-by-side multithreaded
   glider profile now that small-leaf dispatch is active outside single-thread
   mode.
